## 🔑 階段一：準備 4 把金鑰與雲端空間
我們需要準備 4 個關鍵資訊（ID 與 Key），請先將其記錄於記事本備用。

| 項目 | 取得管道 | 說明 |
| :--- | :--- | :--- |
| **LINE_TOKEN** | LINE Developers | 建立 Provider 與 Messaging API Channel，點擊 **Issue** 產生 Token。 |
| **GEMINI_KEY** | Google AI Studio | 點擊 **Get API key**，建立並複製您的 API 金鑰。 |
| **DRIVE_FOLDER_ID** | Google Drive | 建立資料夾，網址 `/folders/` 後的一長串英數字即為 ID。 |
| **SHEET_ID** | Google Sheets | 建立試算表，網址 `/d/` 與 `/edit` 中間的英數字即為 ID。 |

> [!IMPORTANT]
> **試算表設定：** 請在下方手動建立兩個工作表，名稱務必命名為：`會議記錄摘要` 與 `會議記錄逐字稿`。

## 💻 階段二：建立 Google Apps Script (雲端大腦)
1. 前往 [Google Apps Script (GAS)](https://script.google.com/)，點擊「新專案」。
2. 清空編輯器內容，將 `Code.gs` 內的完整程式碼貼上。
3. 在程式碼最上方的 `CONFIG` 區塊，填入您在階段一準備好的 4 把金鑰。
4. 點擊上方的 **「儲存」** (磁碟片圖示)。

## 🌐 階段三：部署為網頁應用程式
1. 在 GAS 編輯器右上角，點擊 **「部署」 > 「新增部署」**。
2. 點擊左側「齒輪 ⚙️」，選擇 **「網頁應用程式」**。
3. **設定如下：**
   - 說明：`AI 會議助手 v1`
   - 執行身分：`我`
   - 誰可以存取：`所有人 (Anyone)`
4. 點擊 **「部署」** 並完成權限授權。
5. **複製** 產生的 「網頁應用程式網址 (URL)」。

## 🔗 階段四：LINE Webhook 串接與收尾
1. **貼上 Webhook 網址：**
   - 回到 LINE Developers > Messaging API 分頁。
   - 在 `Webhook URL` 欄位貼上剛才複製的網址，點擊 **Verify** 確認成功。
   - 開啟 **Use webhook** 開關。
2. **關閉自動回應：**
   - 前往 LINE 官方帳號管理後台 > 回應設定。
   - 將「回應模式」設為「聊天機器人」，並 **停用** 「自動回應訊息」。

## ⚠️ 使用限制與資安注意事項 (免費帳號部署必讀)

本專案使用免費版的 Google 與 LINE 服務即可順利運行，但若要推廣至團隊日常使用，請務必留意以下限制：

### 1. 🛡️ 企業機密與資安隱私 (最重要)
* **限制說明：** 根據 Google 服務條款，使用免費版 Google AI Studio 的 API 金鑰，其傳送內容可能會被 Google 收集作為未來 AI 模型的訓練資料。
* **實務建議：** 若會議內容涉及敏感的人事評議、招募面試或商業機密，**強烈建議使用付費帳號 (Google Cloud 或 Workspace 企業版)** 產生的 API Key，以享有企業級資料隱私保護。

### 2. 💬 LINE 免費推播額度上限
* **限制說明：** LINE 官方帳號免費方案每月僅提供 200 則推播額度。
* **實務建議：** 若會議極度頻繁，請留意額度消耗，避免月底機器人因額度耗盡而無法回應。

### 3. 💾 雲端硬碟容量消耗
* **限制說明：** 免費版帳號總儲存空間為 15 GB。
* **實務建議：** 錄音檔累積極快，建議定期清理舊音檔，或將備份資料夾建立在企業大容量帳號下。

### 4. ⏱️ 系統運算與 API 頻率限制
* **Gemini API：** 免費版每分鐘最多 15 次請求。
* **GAS 限制：** 免費版每天背景執行時間上限為 90 分鐘。
* **建議：** 對個人或單一部門已足夠，但應避免多人在同一時間集體上傳超長音檔。
