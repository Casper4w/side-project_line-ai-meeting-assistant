# side-project_line-ai-meeting-assistant
結合 LINE Bot 與 Gemini API 的自動化會議記錄機器人。

本專案利用 Gemini 2.5 Flash 的多模態處理能力，省去了傳統語音轉文字 (STT) 的中間步驟，直接解析音訊，大幅提升了處理速度與摘要準確度。

## 🔑 階段一：準備 4 把金鑰與雲端空間
我們需要準備 4 個關鍵資訊（ID 與 Key），請先將其記錄於記事本備用。

| 項目 | 取得管道 | 說明 |
| :--- | :--- | :--- |
| **LINE_TOKEN** | LINE Developers | 建立 Provider 與 Messaging API Channel，點擊 **Issue** 產生 Token。 |
| **GEMINI_KEY** | Google AI Studio | 點擊 **Get API key**，建立並複製您的 API 金鑰。 |
| **DRIVE_FOLDER_ID** | Google Drive | 建立資料夾，網址 `/folders/` 後的一長串英數字即為 ID。 |
| **SHEET_ID** | Google Sheets | 建立試算表，網址 `/d/` 與 `/edit` 中間的英數字即為 ID。 |

> [!IMPORTANT]
> **試算表設定：** 請在下方手動建立兩個工作表，名稱務必命名為：`會議記錄摘要` 與 `會議記錄逐字稿`。

## 💻 階段二：建立 Google Apps Script (雲端大腦)
1. 前往 [Google Apps Script (GAS)](https://script.google.com/)，點擊「新專案」。
2. 清空編輯器內容，將 `Code.gs` 內的完整程式碼貼上。
3. 在程式碼最上方的 `CONFIG` 區塊，填入您在階段一準備好的 4 把金鑰。
4. 點擊上方的 **「儲存」** (磁碟片圖示)。

---

## 🌐 階段三：部署為網頁應用程式
1. 在 GAS 編輯器右上角，點擊 **「部署」 > 「新增部署」**。
2. 點擊左側「齒輪 ⚙️」，選擇 **「網頁應用程式」**。
3. **設定如下：**
   - 說明：`AI 會議助手 v1`
   - 執行身分：`我`
   - 誰可以存取：`所有人 (Anyone)`
4. 點擊 **「部署」** 並完成權限授權。
5. **複製** 產生的 「網頁應用程式網址 (URL)」。

---

## 🔗 階段四：LINE Webhook 串接與收尾
1. **貼上 Webhook 網址：**
   - 回到 LINE Developers > Messaging API 分頁。
   - 在 `Webhook URL` 欄位貼上剛才複製的網址，點擊 **Verify** 確認成功。
   - 開啟 **Use webhook** 開關。
2. **關閉自動回應：**
   - 前往 LINE 官方帳號管理後台 > 回應設定。
   - 將「回應模式」設為「聊天機器人」，並 **停用** 「自動回應訊息」。

🎉 **大功告成！現在您可以對著機器人傳送錄音檔進行測試。**

## ⚠️ 使用限制與資安注意事項 (免費帳號部署必讀)

本專案使用免費版的 Google 與 LINE 服務即可順利運行，但若要推廣至團隊日常使用，請務必留意以下幾點先天限制，以確保資料安全與系統穩定：

### 1. 🛡️ 企業機密與資安隱私 (最重要)
* **限制說明：** 根據 Google 服務條款，使用**免費版** Google AI Studio 所產生的 API 金鑰，其傳送的音訊內容與文字，可能會被 Google 收集作為未來 AI 模型的訓練資料。
* **實務建議：** 若會議內容涉及**敏感的人事評議、招募面試紀錄、或是公司內部的製造與營運機密**，強烈建議主管或專案負責人使用已綁定付費帳號的 Google Cloud 專案（或 Workspace 企業版）來產生 API Key，才能享有企業級的資料隱私保護，確保機密絕不外流。

### 2. 💬 LINE 免費推播額度上限
* **限制說明：** LINE 官方帳號的「輕用量（免費）方案」，每個帳號每月僅提供 **200 則**主動推播（Push Message）額度。
* **實務建議：** 機器人在回報進度與傳送摘要時，皆會消耗推播額度。若部門內會議極度頻繁，月底可能會遇到機器人因額度耗盡而「已讀不回」的狀況。屆時可評估由部門申請經費升級 LINE 官方帳號方案。

### 3. 💾 雲端硬碟 (Google Drive) 容量消耗
* **限制說明：** 免費版 Google 帳號的總儲存空間為 15 GB（需與信箱、相簿共用）。
* **實務建議：** 會議錄音檔累積起來容量可觀。若頻繁使用，數月內可能就會將免費空間塞滿。一旦空間不足，程式將無法儲存新錄音檔而引發錯誤。建議定期清理雲端硬碟內的舊音檔，或將備份資料夾建立在公司配發的大容量企業帳號下。

### 4. ⏱️ 系統運算與 API 頻率限制
* **限制說明：** * 免費版 Gemini API 每分鐘最多僅能接收 15 次請求。
  * 免費版 Google Apps Script 每天的背景執行時間上限為 90 分鐘。
* **實務建議：** 對於個人或單一部門日常使用已非常夠用。但若全公司同仁在同一時間（例如週一主管早會結束後）集體上傳兩小時以上的長篇錄音檔，可能會觸發系統的流量防護機制而暫時卡住。
